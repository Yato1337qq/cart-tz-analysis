# Задание 3: Архитектура PUSH-уведомлений

##  Описание

Система должна отправлять PUSH-уведомления пользователям мобильного приложения в зависимости от событий (например: брошенная корзина, статус заказа, рекламные рассылки).

---

##  Основные компоненты

1. **Сервис заказов (Order Service)**
   - генерирует события (создание заказа, отмена, бездействие)

2. **Message Broker (Kafka / RabbitMQ)**
   - принимает события от сервисов
   - обеспечивает асинхронную обработку

3. **Сервис уведомлений (Notification Service)**
   - обрабатывает события
   - формирует текст уведомления
   - определяет получателей

4. **Push-провайдеры**
   - Firebase (Android)
   - APNS (iOS)

---

##  Поток данных

1. Пользователь совершает действие (например, добавил товар в корзину)
2. Сервис заказов отправляет событие
3. Событие попадает в очередь (Message Broker)
4. Сервис уведомлений получает событие
5. Формируется PUSH-уведомление
6. Уведомление отправляется через Firebase/APNS пользователю

---

##  Преимущества решения

- масштабируемость (можно обрабатывать большое количество событий)
- отказоустойчивость (очередь сохраняет события)
- слабая связность сервисов (микросервисная архитектура)

---

##  Примеры уведомлений

- Напоминание о брошенной корзине
- Изменение статуса заказа
- Рекламные уведомления
